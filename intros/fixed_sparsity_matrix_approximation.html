Fixed-sparsity matrix approximation from matrix-vector products

Tyler Chen

Introduction

In this work, we study the task of approximating a d×dd\times dd×d matrix A\mathbf{A}A with a matrix A~\widetilde{\mathbf{A}}A of a sparsity pattern (e.g. diagonal, tridiagonal, block diagonal, etc.). There are many existing linear algebra algorithms for matrices of a given sparsity pattern, so this task yields a proxy for A\mathbf{A}A which is compatible with such algorithms. For example, a banded approximation to A=M−1\mathbf{A} = \mathbf{M}^{-1}A=M−1 could be used as a preconditioned to systems involving M\mathbf{M}M.

The catch is that we assume A\mathbf{A}A can only be accessed using matrix-vector products (called matvecs). That is, we restrict ourselves to algorithms which choose vectors x1,…,xm\mathbf{x}_1, \ldots, \mathbf{x}_mx1​,…,xm​ and subsequently receive responses Ax1,…,Axm\mathbf{A}\mathbf{x}_1, \ldots, \mathbf{A}\mathbf{x}_mAx1​,…,Axm​ from some black-box, but cannot look at A\mathbf{A}A in any other way. We will measure the cost of algorithms by mmm, the total number of matrix-vector queries done, which in many settings may be a realistic proxy for the real-world costs. This is often called the matrix-vector query model.

It’s worth noting that we can recover A\mathbf{A}A in this access model by simply reading off the columns of A\mathbf{A}A one-by-one. This means we can effectively run any classical algorithm for the cost of ddd matvecs. If ddd is very big this is not practical, but it is worth keeping in mind as a trivial upper bound for the cost of any algorithm in this model. In general, our goal will be to design algorithms which use m≪dm\ll dm≪d matvecs.

One of the most powerful features of the matvec query model is that it is often possible to prove lower-bounds for the hardness of a given task. That is, to show that it is impossible (for any algorithm) to solve the task unless the algorithm uses at least a certain number of queries. This is exciting, because it offers a way of understanding the “difficulty” of linear algebra tasks! In contrast, proving lower-bounds on the number of floating point operations (which has classically been the measure of cost for linear-algebra algorithms), is seemingly far beyond current techniques.

One thing I think is really cool about this paper, is that almost everything is self-contained! In particular, besides a few well-known results on properties of Gaussian matrices and a qualitative central limit theorem called the Berry-Esseen theorem, everything needed is proved within the paper using relatively elementary techniques which would be understandable to anyone with an undergraduate-level knowledge of linear-algebra and probability. In fact, the general technique is so simple that we even prove a lower-bound for diagonal estimation on this page! Overall, I hope that this paper can serve as a case-study on the power and beauty of the matrix-vector query model.

Two theoretical problems

For convenience, we will measure the error in the Frobenius norm: ∥A−A~∥F=∑i,j([A] i,j−[A~] i,j)2. \|\mathbf{A} - \widetilde{\mathbf{A}} \|_{\mathsf{F}} = \sqrt{\sum_{i,j} \big([\mathbf{A}]_{\,i,j} - [\widetilde{\mathbf{A}}]_{\,i,j} \big)^2}. ∥A−A∥F​=i,j∑​([A]i,j​−[A]i,j​)2​. Let’s also encode the desired sparsity pattern in a binary matrix S\mathbf{S}S. The nonzero entries of S\mathbf{S}S will tell us where A~\widetilde{\mathbf{A}}A is allowed to have a nonzero entries. It is pretty clear then that the best sparse approximation to A\mathbf{A}A is simply S∘A\mathbf{S}\circ \mathbf{A}S∘A, where “∘\circ∘” is the Hadamard (entrywise) product. That is, the best approximation is obtained by zeroing out the entries of A\mathbf{A}A which are required to be zero.

We will study the following problems:

Problem 1 (Best approximation by a matrix of fixed sparsity). Given a matrix A∈Rn×d\mathbf{A} \in\mathbb{R}^{n\times d}A∈Rn×d and a binary matrix S∈{0,1}n×d\mathbf{S} \in \{0,1\}^{n\times d}S∈{0,1}n×d, find a matrix A~\widetilde{\mathbf{A}}A so that A~=S∘A~\widetilde{\mathbf{A}} = \mathbf{S}\circ\widetilde{\mathbf{A}}A=S∘A and ∥A−A~∥F≤(1+ε) ∥A−S∘A∥F. \| \mathbf{A} - \widetilde{\mathbf{A}} \|_\mathsf{F} \leq (1+\varepsilon) \, \| \mathbf{A} - \mathbf{S}\circ \mathbf{A} \|_\mathsf{F}. ∥A−A∥F​≤(1+ε)∥A−S∘A∥F​.

Problem 2 (Best approximation to on-sparsity-pattern entries). Given a matrix A∈Rn×d\mathbf{A} \in\mathbb{R}^{n\times d}A∈Rn×d and a binary matrix S∈{0,1}n×d\mathbf{S} \in \{0,1\}^{n\times d}S∈{0,1}n×d, find a matrix A~\widetilde{\mathbf{A}}A so that A~=S∘A~\widetilde{\mathbf{A}} = \mathbf{S}\circ\widetilde{\mathbf{A}}A=S∘A and ∥S∘A−A~∥F2≤ε ∥A−S∘A∥F2. \| \mathbf{S}\circ \mathbf{A} - \widetilde{\mathbf{A}} \|_\mathsf{F}^2 \leq \varepsilon \,\| \mathbf{A} - \mathbf{S}\circ \mathbf{A} \|_\mathsf{F}^2. ∥S∘A−A∥F2​≤ε∥A−S∘A∥F2​.

Note that since A~\widetilde{\mathbf{A}}A has the same sparsity as S\mathbf{S}S; i.e. A~=S∘A~\widetilde{\mathbf{A}} = \mathbf{S}\circ\widetilde{\mathbf{A}}A=S∘A, ∥A−A~∥F2=∥A−S∘A∥F2+∥S∘A−A~∥F2. \| \mathbf{A} - \widetilde{\mathbf{A}} \|_\mathsf{F}^2 = \| \mathbf{A} - \mathbf{S}\circ \mathbf{A} \|_\mathsf{F}^2 + \|\mathbf{S}\circ\mathbf{A} - \widetilde{\mathbf{A}}\|_\mathsf{F}^2. ∥A−A∥F2​=∥A−S∘A∥F2​+∥S∘A−A∥F2​. This allows us to relate Problems 1 and 2, and for ε∈(0,1)\varepsilon \in (0,1)ε∈(0,1), the two are equivalent up to (small) constants.

A simple algorithm

When the sparsity pattern S\mathbf{S}S has at most sss non-zero entries per row, this algorithm uses m=O(s/ε)m = O(s/\varepsilon)m=O(s/ε) non-adaptive matrix-vector product queries. Specifically, the algorithm computes Z=AG\mathbf{Z} = \mathbf{A}\mathbf{G}Z=AG, where G\mathbf{G}G is a d×md\times md×m matrix with independent standard normal entries, and then outputs the matrix A~=argmin⁡X=S∘X∥Z−XG∥F. \widetilde{\mathbf{A}} = \operatornamewithlimits{argmin}_{\mathbf{X} = \mathbf{S}\circ\mathbf{X}} \| \mathbf{Z} - \mathbf{X} \mathbf{G}\|_\mathsf{F}. A=X=S∘Xargmin​∥Z−XG∥F​. It’s not hard to see how to implement this algorithm. First, observe that each row of the approximation can be computed independently. Then, note that the solution to each row is obtained by solving a standard least squares problem involving the relevant rows of G\mathbf{G}G.

Using standard tools from high dimensional probability, which characterize the expected Frobenius norm of the pseudoinverse of a Gaussian matrix, it is relatively simple to establish the following convergence guarantee:

Theorem 1. Consider any A∈Rn×d\mathbf{A} \in\mathbb{R}^{n\times d}A∈Rn×d and any S∈{0,1}n×d\mathbf{S}\in\{0,1\}^{n\times d}S∈{0,1}n×d with at most sss nonzero entries per row. Then, for any m≥s+2m\geq s+2m≥s+2, using mmm randomized matrix-vector queries, the matrix A~\widetilde{\mathbf{A}}A is equal to S∘A\mathbf{S}\circ\mathbf{A}S∘A in expectation and satisfies E[∥S∘A−A~∥F2]≤sm−s−1∥A−S∘A∥F2. \mathbb{E}\Bigl[\|\mathbf{S}\circ\mathbf{A} - \widetilde{\mathbf{A}}\|_\mathsf{F}^2 \Bigr] \leq {\frac{s}{m-s-1}} \| \mathbf{A} - \mathbf{S}\circ \mathbf{A} \|_\mathsf{F}^2. E[∥S∘A−A∥F2​]≤m−s−1s​∥A−S∘A∥F2​. The above inequality is equality if each row of S\mathbf{S}S has exactly sss non-zero entries.

If we set m=O(s/ε)m = O(s/\varepsilon)m=O(s/ε), then we solve Problem 2 (and hence Problem 1), at least in expectation. One can then use Markov’s inequality to get a probability bound. This matches the gurantees for a number of existing algorithms for special cases of Problem 1 and 2 [1], [2], [3], [4], etc., and we comment on the connections in more detail in the paper.

A lower bound

For any fixed A\mathbf{A}A, there is an algorithm which solves Problem 1 exactly using zero queries; simply hard-code the algorithm to output S∘A\mathbf{S}\circ\mathbf{A}S∘A. This is not a practically useful algorithm, because it only works for the one particular A\mathbf{A}A and we would be unable to practically ``find’’ such an algorithm for a given matrix. However, it means it’s impossible to prove a statement like: “For any matrix A\mathbf{A}A, no algorithm using (some amount of queries depending on ε\varepsilonε) can solve Problem 1”. Instead, it is standard to consider a distribution of matrices A\mathbf{A}A (or equivalently, a random matrix). While an algorithm might be hard-coded for the particular distribution we consider, there is hope that the distribution is always still very random, even conditioned on a small number of arbitrary matvec queries.

In particular, in the paper we prove a precise version of the following:

Theorem 2. There are constants c,C>0c,C>0c,C>0 so that, for any ε∈(0,c)\varepsilon\in(0,c)ε∈(0,c) and s≥1s\geq 1s≥1, there is a distribution on matrices A\mathbf{A}A such that, for any sparsity pattern S\mathbf{S}S in a broad class of sparsity patterns, and for any algorithm that uses m≤Cs/εm\leq Cs/\varepsilonm≤Cs/ε matrix-vector queries to A\mathbf{A}A to output a matrix A~\widetilde{\mathbf{A}}A, P[∥A−A~∥F≤(1+ε) ∥A−S∘A∥F]≤125. \mathbb{P}\Bigl[ \| \mathbf{A} - \widetilde{\mathbf{A}} \|_\mathsf{F} \leq (1+\varepsilon) \,\| \mathbf{A} - \mathbf{S}\circ \mathbf{A} \|_\mathsf{F}\Bigr] \leq \frac{1}{25}. P[∥A−A∥F​≤(1+ε)∥A−S∘A∥F​]≤251​.

This shows that there are problems for which no algorithm can use more than a constant (e.g. 100100100) times fewer matvecs than the algorithm analyzed in the previous section.

At a high-level, the lower-bound technique is pretty simple. The basic idea is that if A=GTG\mathbf{A} = \mathbf{G}^\mathsf{T}\mathbf{G}A=GTG, where G\mathbf{G}G is d×dd\times dd×d random Gaussian matrix, then after a small number of (possibly adaptive) matvec queries to A\mathbf{A}A, there is still too much randomness in A\mathbf{A}A (conditioned on the information leared by the queries) to learn what we need [5].

What changes in the general case?

The above proof actually captures pretty much all of the important ideas we use for the lower-bound provided in the paper. There are a number of details which differ. The most important are perhaps the following:

  • In the above proof, the entries of d\mathbf{d}d were independent. For other sparsity patterns this won’t be the case. While we used independence in our computation of the variance (which was just for intuition), we did not use independence in the actual proof.
  • In the paper, we use the distribution GTG\mathbf{G}^\mathsf{T}\mathbf{G}GTG. This somewhat complicates the analog of Lemma 1, as well as computations (since things aren’t all Gaussian anymore). However, it gives a lower-bound against the larger class of algorithms which can perform matvecs with the transpose.
  • In the paper we are more careful about saying things like “let d=1/εd = 1/\varepsilond=1/ε” and “d=0\mathbf{d} = \mathbf{0}d=0 is the best choice”, and handle these cases explicitly / precisely.
