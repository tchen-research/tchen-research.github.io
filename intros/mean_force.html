Introduction

A quantum system is described by a Hamiltonian matrix H\mathbf{H}H. When the system is at thermal equilibrium at temperature 1/β1/\beta1/β, then the state of the system is described by the matrix ρ=exp⁡(−βH)/Z(β),Z(β)=tr⁡(exp⁡(−βH)). \bm{\rho} = \exp(-\beta \mathbf{H}) / Z(\beta) ,\qquad Z(\beta) = \operatorname{tr}(\exp(-\beta \mathbf{H})). ρ=exp(−βH)/Z(β),Z(β)=tr(exp(−βH)). The function Z(β)Z(\beta)Z(β) is called the partition function, and gives us access to all kinds of thermodynamic quantities such as the energy, specific heat, magnetization, etc.

Often we are interested in the state of some subsystem of the total system. This is represented by ρ∗(β)=tr⁡b(−βH)/Z(β), \bm{\rho}^*(\beta) = \operatorname{tr}_{\mathrm{b}}(-\beta \mathbf{H}) / Z(\beta), ρ∗(β)=trb​(−βH)/Z(β), where tr⁡b(⋅)\operatorname{tr}_{\mathrm{b}}(\cdot)trb​(⋅) is the partial trace described below.

Exponentially big linear algebra

Computing the trace (and partial trace) of a given matrix is trivial. For the trace you just sum the diagonal entries, and for the partial trace you do something similarly easily. However, we are starting with H\mathbf{H}H not exp⁡(−βH)\exp(-\beta \mathbf{H})exp(−βH), and this presents some issues.

For a system with NNN sites, the size of H\mathbf{H}H is 2N2^N2N. This means that even for reasonably small NNN, storing a N×NN\times NN×N matrix is impossible. For instance, if N=20N=20N=20, then 2N2^N2N is about one million, and storing a N×NN\times NN×N matrix of double precision floating point numbers (64 bits per number) would require over 8 terrabtyes of memory! Increasing NNN by one quadruples the memory costs, so even if we have 8 terrabytes of memory, we’ll quickly run out for NNN even just a bit larger.

Fortunately, H\mathbf{H}H typically has a very sparse representation, often with just O(N)O(N)O(N) nonzero entries. While this means we can store H\mathbf{H}H itself, exp⁡(−βH)\exp(-\beta \mathbf{H})exp(−βH) is typically not sparse and therefore it is intractable to store. Moreover, even if we would store exp⁡(−βH)\exp(-\beta \mathbf{H})exp(−βH), computing it would be a big task!

Stochastic trace estimation

There are lots of algorithms for estimating vTf(H)v\mathbf{v}^\mathsf{T}f(\mathbf{H})\mathbf{v}vTf(H)v using only matrix-vector products with H\mathbf{H}H. For instance, the Lanczos method for matrix function approximation (Lanczos-FA) is a common choice.

There are lots of nice overviews on stochastic trace estimation, and this blog post by Ethan Epperly is especially good. For our purposes, we only need to know that if v\mathbf{v}v is a vector for which E[vvT]=I\mathbb{E}[\mathbf{v} \mathbf{v}^\mathsf{T}] = \mathbf{I}E[vvT]=I (i.e. all the entries are uncorrelated), then E[vTAv]=tr⁡(A). \mathbb{E}\big[\mathbf{v}^\mathsf{T}\mathbf{A} \mathbf{v}\big] = \operatorname{tr}(\mathbf{A}). E[vTAv]=tr(A). A simple way to generate such a v\mathbf{v}v is by taking each entry to be an independent standard normal random variable. We will call estimators of the form vTAv\mathbf{v}^\mathsf{T}\mathbf{A}\mathbf{v}vTAv a quadratic trace estimator.
